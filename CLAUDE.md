# strus — Agent Context

Polish morphological spaced repetition system. Users add **lemmas** (dictionary citation forms,
e.g. "dom", "iść"); Morfeusz2 generates all inflected **word forms**; FSRS schedules one
learning card per form.

---

## Repository Layout

```
strus/
├── packages/
│   ├── morph/   @strus/morph   — Morfeusz2 CLI subprocess wrapper + tag parser
│   ├── core/    @strus/core    — FSRS scheduler, domain types (Lemma, LearningTarget, …)
│   ├── db/      @strus/db      — Drizzle ORM schema, migrations, db client (bun:sqlite)
│   ├── api/     @strus/api     — Elysia + oRPC HTTP server + OpenAPI spec generation
│   └── cli/     @strus/cli     — Commander CLI (thin wrapper over the HTTP API)
├── CLAUDE.md
├── pnpm-workspace.yaml
└── package.json
```

Package dependency order (bottom → top): `morph` → `core` → `db` → `api` / `cli`

## Common Commands

```bash
# From repo root
pnpm typecheck                 # typecheck all packages
pnpm -r exec tsc --noEmit     # same, more explicit

# Dev server — IMPORTANT: kill any running instance first!
# Bun enables SO_REUSEPORT by default on Linux, so a second server silently shares
# port 3457, causing requests to load-balance between two separate DB connections
# (intermittent data, split-brain "not found" errors).
pkill -f 'bun run.*src/index.ts' 2>/dev/null; sleep 1
STRUS_DB_PATH=/absolute/path/to/strus.db PORT=3457 bun run src/index.ts
# Or with watch mode:
STRUS_DB_PATH=/absolute/path/to/strus.db PORT=3457 bun run --watch src/index.ts

# Schema changes → generate migration
pnpm --filter @strus/db run generate    # interactive: drizzle-kit generate
# (drizzle-kit will ask about renames — answer them interactively via PTY)

# Write openapi.json to disk
pnpm --filter @strus/api run generate:spec

# CLI (from packages/cli/)
bun run src/index.ts --help
bun run src/index.ts lemma add dom --pos subst
bun run src/index.ts quiz
```

## Data Model

```
VocabList          — named collection of notes
  └─ VocabListNote   (join table: listId, noteId)
Note               — kind = "morph" | "gloss" | "basic"
  ├─ kind=morph:  backed by a Lemma; no front/back; generates morph_form Cards automatically
  ├─ kind=gloss:  backed by a Lemma + back (translation); generates gloss_forward/reverse Cards
  └─ kind=basic:  standalone; front + back text; generates one basic_forward Card
Lemma              — citation/dictionary form; source = "morfeusz" | "manual"
  └─ MorphForm       — individual inflected form (orth + NKJP tag)
Card               — FSRS card state; kind = morph_form | gloss_forward | gloss_reverse | basic_forward
  └─ Review          — immutable review history record
```

### Lemma.source

- `"morfeusz"` (default): word forms are auto-generated by Morfeusz2 at insert time
- `"manual"`: no auto-generation; forms are user-supplied

### NKJP Tags

Morfeusz2 uses the NKJP tagset. Examples:
- `subst:sg:nom:m3` — noun, singular, nominative, inanimate masculine
- `subst:sg:inst:m3` — noun, singular, instrumental, inanimate masculine
- `verb:inf:imperf` — verb, infinitive, imperfective

## API Layer (oRPC)

All procedures use the **builder pattern**:

```ts
const myProc = os
  .route({ method: "GET", path: "/things", tags: ["Things"], summary: "…" })
  .input(z.object({ id: z.string().uuid().optional() }))
  .output(ThingOutput)
  .handler(async ({ input }) => { … });
```

The router is a plain nested object:

```ts
export const router = {
  lemmas: { list: lemmasList, create: lemmasCreate, … },
  session: { due: sessionDue, review: sessionReview },
};
```

The oRPC handler is mounted on Elysia via `OpenAPIHandler` from `@orpc/openapi/fetch`:

```ts
const orpcHandler = new OpenAPIHandler(router);
app.all("/api/*", async ({ request }) => {
  const result = await orpcHandler.handle(request, { prefix: "/api" });
  return result.matched ? result.response : new Response("Not found", { status: 404 });
});
```

> **No `@orpc/elysia` package exists.** Use the generic fetch adapter.

The OpenAPI spec is generated at server startup via `OpenAPIGenerator` + `ZodToJsonSchemaConverter` and served at `GET /openapi.json`. Swagger UI (CDN) is at `GET /docs`.

## Database (Drizzle + bun:sqlite)

**Adapter:** `drizzle-orm/bun-sqlite` (NOT `drizzle-orm/better-sqlite3`)

```ts
import Database from "bun:sqlite";
import { drizzle } from "drizzle-orm/bun-sqlite";

const sqlite = new Database(process.env.STRUS_DB_PATH);
export const db = drizzle(sqlite, { schema });
```

**Migrations** are applied at API startup (not via CLI):

```ts
import { migrate } from "drizzle-orm/bun-sqlite/migrator";
migrate(db, { migrationsFolder: resolve(import.meta.dir, "../../db/migrations") });
```

`drizzle-kit` is a **dev-only** tool — only used to generate SQL migration files. It runs under
Node (not Bun). `better-sqlite3` is kept as a `devDependency` of `@strus/db` solely for
drizzle-kit's use; it is never imported at runtime.

**Queries are synchronous** with `bun:sqlite`. Don't `await` synchronous query methods:

```ts
// ✓ correct
const rows = db.select().from(lemmas).all();

// ✗ wrong (pointless await, but harmless)
const rows = await db.select().from(lemmas).all();
```

## FSRS / core

ts-fsrs **v5** API — key points:

- `Card` shape requires `learning_steps: number` (new in v5; we default to `0` since it's not
  persisted in the DB yet)
- `Rating.Manual = 0` is excluded from `RecordLog`; only ratings 1–4 are scored
- `createEmptyCard()` returns a card with `last_review` omitted (not `undefined`)
- With `exactOptionalPropertyTypes: true`, use conditional spreads for optional fields:
  ```ts
  // ✓ correct
  const card = { ...base, ...(val !== undefined ? { lastReview: val } : {}) };

  // ✗ wrong — assigns undefined, violates exactOptionalPropertyTypes
  const card = { ...base, lastReview: val };
  ```

## Morfeusz2

Morfeusz2 is invoked as a **CLI subprocess**. The `@strus/morph` package
spawns it via `Bun.spawn`, writes the lemma to stdin, and parses tab-separated output lines:

```
orth\tlemma\ttag
dom\tdom\tsubst:sg:nom:m3
domu\tdom\tsubst:sg:gen:m3
…
```

Morfeusz2 is optional at runtime — if the binary is unavailable, `generate()` throws and the
API gracefully skips form generation (logs a warning, returns the lemma without forms).

## TypeScript Conventions

- **`exactOptionalPropertyTypes: true`** is set in all tsconfigs — this is stricter than
  standard TypeScript. Optional properties cannot be assigned `undefined` explicitly.
- All packages use `"moduleResolution": "bundler"` (Bun-compatible).
- Source files are run directly by Bun; `tsc` is used only for type-checking (`--noEmit`).
  There is no build/emit step.
- Workspace packages import each other using the `@strus/*` package name, not relative paths
  across package boundaries.
- Zod v3 only. If you see `@orpc/zod/zod4`, that's for Zod v4 and is **wrong** for this project.

## Adding a New API Procedure

1. Define Zod input/output schemas in `packages/api/src/router.ts`
2. Add the `os.route(…).input(…).output(…).handler(…)` procedure
3. Add it to the `router` export object
4. Run `pnpm typecheck` — the OpenAPI spec auto-regenerates at next server startup

## Changing the DB Schema

1. Edit `packages/db/src/schema.ts`
2. `pnpm --filter @strus/db run generate` (interactive — answer rename prompts carefully)
3. Inspect the generated SQL in `packages/db/migrations/`
4. Update any affected TypeScript types in `packages/core/src/types.ts`
5. `pnpm typecheck` to verify
6. Restart the server — migrations apply automatically at startup
